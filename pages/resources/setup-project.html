⚙️ Standalone TypeScript Project Setup

For a standalone TypeScript project (one that will simply be executed from the command-line), you can simplify the project setup steps and run your TypeScript code using ts-node (instead of node) without transpiling your code to JavaScript.

Prerequisites

  • npm is already installed on your computer

Setup Steps

Follow these steps to set up a new TypeScript project:

  1. Create a directory for your project
  2. CD into the new directory
  3. Run npm init -y to initialize a new Node.js project
  4. Run npm install typescript --save-dev to install TypeScript
  5. Run npm install @types/node --save to install Node type definitions
  6. Run npm install ts-node --save-dev to install ts-node
  7. Create an src directory
  8. Create a TypeScript file in your src directory and write your TypeScript code
  9. Run npx tsc --init to create a tsconfig.json file
  10. Edit compilerOptions in tsconfig.json to contain the following settings:

Example: tsconfig.json Configuration

{
  "compilerOptions": {
    "target": "ESNext",
    "module": "commonjs",
    "sourceMap": true,
    "outDir": "dist"
  }
}
  1. Add other compilerOptions settings as desired

💡 Note

If you do not include these configuration steps, ts-node will use reasonable defaults when running your code.

Running Your TypeScript Code

Once your project is set up, you can run your TypeScript files directly using ts-node:

Command to Run TypeScript Files

npx ts-node src/<Your Typescript File.ts>

Debugging in VS Code

You can set up a standalone TypeScript program for debugging in VS Code by following these steps:

  1. Select Run → Start Debugging
  2. Under "Run and Debug", click "create a launch.json file"
  3. Replace the contents of the launch.json file with the following:

Example: launch.json Configuration

{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Debug <Name of Program>",
      "runtimeArgs": ["-r", "ts-node/register"],
      "args": ["${workspaceFolder}/src/<File to Execute>.ts"],
      "sourceMaps": true
    }
  ]
}

💡 Configuration Notes

  • Change the name attribute value to a name that describes your program
  • Change <File to Execute> inside the args attribute value with the name of the file you want to debug
  • You can have multiple configurations inside the configurations attribute array value. This is useful if your project contains multiple TypeScript programs.

Quick Reference

All Setup Commands in Order

# Create and navigate to project directory
mkdir my-typescript-project
cd my-typescript-project

# Initialize npm and install dependencies
npm init -y
npm install typescript --save-dev
npm install @types/node --save
npm install ts-node --save-dev

# Create source directory and initialize TypeScript
mkdir src
npx tsc --init

# Run a TypeScript file
npx ts-node src/index.ts
